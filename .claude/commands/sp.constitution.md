# Constitution Command

Access and update the project constitution.

## Usage
View or modify the project's core principles and guidelines.

## Constitution Location
`.specify/memory/constitution.md`

## What's In The Constitution
- **Purpose**: Why this project exists
- **Core Principles**: Foundational beliefs
- **Development Philosophy**: How we build
- **Quality Standards**: What we demand
- **Community Values**: How we work together

## When To Update
- Adding major new features
- Changing project direction
- Learning important lessons
- Adapting to new needs

## Update Process
1. Read current constitution
2. Propose changes with rationale
3. Discuss with stakeholders
4. Document in constitution
5. Announce to team

## Example Updates
- "We now prioritize accessibility"
- "We've adopted TypeScript for safety"
- "We require 80% test coverage"
- "We use conventional commits"

## Constitution Guides
- Feature decisions
- Code reviews
- Architecture choices
- Conflict resolution

## Related Commands
- `sp.adr` - Document architectural decisions
- `sp.phr` - Record implementation lessons
