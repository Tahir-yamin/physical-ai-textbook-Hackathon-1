# Analyze Command

Analyze existing code, specs, or implementations for consistency and quality.

## Usage
Review artifacts for completeness, consistency, and potential issues.

## What To Analyze

### 1. Specifications
- Are requirements clear?
- Any contradictions?
- Missing edge cases?
- Testable criteria?

### 2. Code
- Follows style guide?
- Proper error handling?
- Adequate test coverage?
- Performance concerns?
- Security vulnerabilities?

### 3. Architecture
- Consistent patterns?
- Appropriate abstractions?
- Scalability considered?
- Dependencies managed?

### 4. Cross-Artifact Consistency
- Spec matches implementation?
- Tasks align with plan?
- Documentation up to date?

## Analysis Report Structure
```
specs/[feature]/analysis-report.md

## Summary
## Findings
### Critical Issues
### Warnings
### Suggestions
## Recommendations
## Action Items
```

## Analysis Checklist
- [ ] Spec reviewed
- [ ] Plan reviewed
- [ ] Tasks reviewed
- [ ] Code reviewed
- [ ] Tests reviewed
- [ ] Docs reviewed
- [ ] Consistency checked
- [ ] Report generated

## Next Steps
1. Address critical issues
2. Fix warnings
3. Consider suggestions
4. Update artifacts as needed
